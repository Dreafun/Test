# ✨ Ночной путь к Лунной библиотеке

> Некоторые двери открываются не ключом, а правильным настроением.

## Введение

Лунная библиотека появляется только после полуночи.  
Днём это просто старое здание из серого камня, а ночью её окна начинают светиться мягким серебряным светом, будто внутри не лампы, а осколки звёзд.

Эта статья — просто пример того, **как может выглядеть гайд-статья** на твоём сайте.

!!! note "Небольшая ремарка"
    Тут специально смешаны атмосфера и структура, чтобы текст выглядел не как сухая инструкция, а как статья.

## Что взять с собой

- фонарь
- тёплую кофту
- записную книжку
- что-нибудь сладкое
- спокойную голову

## Короткий маршрут

1. Подойти к северной ограде.
2. Дождаться, пока на башне пробьёт полночь.
3. Найти дверь без ручки.
4. Коснуться камня рядом с аркой.
5. Войти только после второго щелчка замка.

!!! warning "Не спеши"
    Если дверь открылась сразу — это, скорее всего, не та дверь.

## Что можно встретить внутри

| Зона | Что там есть | Риск |
|------|---------------|------|
| Главный зал | Каталоги и карты | Низкий |
| Восточное крыло | Редкие книги | Средний |
| Нижний архив | Запретные рукописи | Высокий |

## Советы для разных случаев

=== "Если ты там впервые"

    Не иди слишком быстро.  
    В незнакомом месте лучше сначала уловить ритм пространства.

=== "Если ты пишешь гайд"

    Отмечай не только действия, но и ощущения: свет, звук, запах, детали.  
    Именно они делают статью живой.

=== "Если ты просто читатель"

    Смотри на структуру: заголовки, блоки, таблицы, цитаты.  
    Они помогают не потеряться в длинном тексте.

## Сворачиваемый фрагмент

??? tip "Записка старого библиотекаря"
    Если книга лежит открытой на столе, не думай, что она ждёт именно тебя.  
    Иногда это просто предупреждение для следующего человека.

## Маленькая врезка с акцентом

Можно выделять ==важные слова==, делать текст **жирным**, *курсивным* и `моноширинным`.

Также работают:
- задачи
- [x] фонарь взят
- [x] карта найдена
- [ ] здравый смысл не потерян

## Блок кода

```bash
mkdocs serve
mkdocs build
```

## Итог

Такой формат уже похож не на документацию, а на **свой аккуратный сайт со статьями**.  
То есть ты можешь делать:
- гайды
- подборки
- тематические статьи
- атмосферные тексты с навигацией
